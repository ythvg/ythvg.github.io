场景二：异步竞态问题
对话示例 · 并发 Bug
你：高并发压测时，库存会超卖，但单次测试完全正常。
这是库存扣减的代码：

async function deductStock(productId: string, qty: number) {
  const product = await Product.findById(productId);
  if (product.stock < qty) throw new Error('库存不足');
  await Product.update({ stock: product.stock - qty });
}

Claude：
这是经典的 TOCTOU（检查时间与使用时间不一致）竞态问题。

并发场景：两个请求同时读到 stock=1，都通过了检查，
都执行了扣减，结果 stock 变成 -1。

方案一：数据库原子更新（推荐）
  await Product.update(
    { stock: { decrement: qty } },
    { where: { id: productId, stock: { gte: qty } } }
  );
  // 利用数据库行锁，只有 stock >= qty 时才更新
  // 检查影响行数：0 表示库存不足

方案二：Redis 分布式锁
  const lock = await redisLock.acquire(`stock:${productId}`);
  try {
    // ... 串行执行检查+扣减
  } finally {
    await lock.release();
  }

方案一性能更好，帮你实现？

TypeScript · productService.ts（关键片段） 
interface ProductQuery {
  page: number;
  pageSize: number;
  category?: string;
  minPrice?: number;
  maxPrice?: number;
  inStock?: boolean;
  sortBy?: 'price' | 'createdAt';
  order?: 'asc' | 'desc';
}

async function listProducts(q: ProductQuery) {
  const where: Prisma.ProductWhereInput = {};
  if (q.category) where.category = q.category;
  if (q.inStock !== undefined) where.stock = q.inStock ? { gt: 0 } : 0;
  if (q.minPrice || q.maxPrice) {
    where.price = {};
    if (q.minPrice) where.price.gte = q.minPrice;
    if (q.maxPrice) where.price.lte = q.maxPrice;
  }

  const skip = (q.page - 1) * q.pageSize;
  const [data, total] = await prisma.$transaction([
    prisma.product.findMany({
      where, skip, take: q.pageSize,
      orderBy: { [q.sortBy ?? 'createdAt']: q.order ?? 'desc' },
    }),
    prisma.product.count({ where }),
  ]);

  return {
    data, total, page: q.page,
    pageSize: q.pageSize,
    totalPages: Math.ceil(total / q.pageSize),
  };
}

这是「XX 电商平台」的后端 API 服务，Node.js + TypeScript。

- 数据库：PostgreSQL 15，ORM 用 Prisma
- 缓存：Redis 7，客户端用 ioredis
- 测试：Vitest + Supertest，运行命令：npm test

Git Pre-commit Hook：自动 Review
.git/hooks/pre-commit 
#!/bin/bash
# 提交前自动检查安全问题

DIFF=$(git diff --staged)
if [ -z "$DIFF" ]; then
  exit 0
fi

echo "🔍 Claude Code 正在检查提交..."

RESULT=$(echo "$DIFF" | claude -p "
检查这个 diff 是否有以下问题：
1. 硬编码的密钥、密码、Token
2. console.log 调试代码残留
3. TODO/FIXME 注释未处理
4. 明显的安全漏洞（SQL 注入、XSS 等）

如果发现问题，输出具体位置和说明。
如果没有问题，只输出：CLEAN
")

if [ "$RESULT" != "CLEAN" ]; then
  echo "⚠️  发现问题，请修复后再提交："
  echo "$RESULT"
  exit 1
fi

echo "✅ 检查通过"
GitHub Actions 集成
.github/workflows/claude-review.yml 
name: Claude Code Review
on:
  pull_request:
    types: [opened, synchronize]

jobs:
  review:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
        with:
          fetch-depth: 0

      - name: Install Claude Code
        run: npm install -g @anthropic-ai/claude-code

      - name: AI Code Review
        env:
          ANTHROPIC_API_KEY: ${{ secrets.ANTHROPIC_API_KEY }}
        run: |
          git diff origin/main...HEAD | claude -p "
            对这个 PR 的代码变更进行 review，关注：
            1. 潜在的 bug 和边界情况
            2. 安全漏洞
            3. 性能问题
            4. 代码风格和可维护性
            输出 Markdown 格式
          " > review.md
          cat review.md

      - name: Post Review Comment
        uses: actions/github-script@v7
        with:
          script: |
            const fs = require('fs');
            const body = fs.readFileSync('review.md', 'utf8');
            github.rest.issues.createComment({
              ...context.repo,
              issue_number: context.issue.number,
              body: `## 🤖 Claude Code Review\n\n${body}`
            });
ℹ️ 在 CI 中使用 API Key 时，需要在 GitHub Secrets 中配置 ANTHROPIC_API_KEY，并注意 API 调用费用（建议只在关键分支的 PR 触发）。

使用技巧

高效使用 10 条建议

掌握这些技巧，让 Claude Code 发挥出最大价值。

🎯
描述目标，而非步骤
说"给用户接口加分页"，不要说"修改第 45 行加 limit 参数"。让 Claude 自主决定怎么做。
📋
粘贴完整错误信息
把完整的错误堆栈、日志直接粘进对话，包含的上下文越多，定位越准确。
🧪
要求一并更新测试
每次要求新功能时加上"同步更新测试"，确保改动有测试覆盖。
💾
用 CLAUDE.md 写项目规范
把代码风格、禁止事项、常用命令写进 CLAUDE.md，每次启动都自动生效。
🔄
迭代，不要一次性
先做基础版本，确认没问题再加细节。避免一条指令包含太多要求。
🚫
明确说禁止事项
"不要引入新依赖"、"保持 API 兼容性"、"不修改 README"——限制越清晰越好。
🗂️
大任务先规划
复杂任务先说"帮我列一下实施步骤"，确认方向再执行，避免返工。
🌐
充分利用 Web 搜索
遇到你不确定的包版本、API 文档，直接叫 Claude 去搜最新资料。
🧠
用"记住"写入长期记忆
说"记住：我们用 pnpm 不用 npm"，下次对话 Claude 会记得。
⚡
用 -p 做快速单次查询
claude -p "解释这个函数" 快速得到答案，不用进入交互模式。
