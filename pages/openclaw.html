自主任务执行

自然语言下指令，智能体不仅回答，还能主动操作系统、访问网页、处理邮件、整理文件、自动编写并运行代码，完成完整工程任务。

💬

多平台消息接入

支持通过 Telegram、WhatsApp、Signal、Discord、Slack 发送指令。你的手机聊天窗口就是 AI 的控制台。

🧠

持久化记忆系统

跨会话记录工作习惯、项目细节、用户偏好。每日自动写日记、更新身份文件、维护待办清单，越用越懂你。

每 30 分钟主动检查待办事项，无需用户干预即可执行定期任务：舆情监控、邮件归档、数据统计报告……

TypeScript CLI 架构

核心是 TypeScript 编写的命令行应用，轻量独立、易于二次开发，跨平台支持 macOS / Linux / Windows。

工作原理

OpenClaw 如何运行？

四步理解 OpenClaw 的执行机制——从指令接收到任务完成。

STEP 1

指令接收

你通过 Telegram、WhatsApp 等消息平台发送自然语言指令，或通过配置文件设置定时触发条件。

STEP 2

意图理解

OpenClaw 调用配置的大语言模型（Claude / GPT-4o 等）解析指令，拆解为可执行的子任务序列。

STEP 3

工具调用

智能体调用内置工具（Shell 执行器、浏览器自动化、邮件 API、文件系统等）或已安装的 Skill 插件，按序执行子任务。

STEP 4

结果汇报

任务完成后通过消息平台回报结果，同时将本次执行摘要写入本地记忆文件，供下次任务参考。

❤️

Heartbeat 心跳机制

OpenClaw 在后台持续运行一个 Heartbeat 调度器，每 30 分钟自动检查：是否有待处理的定时任务、是否有需要跟进的事项、是否需要更新记忆摘要。即使你没有主动发送消息，它也在工作。

安全须知

强大能力需要
负责任的配置

给 AI 赋予 Shell 权限本质上是"上帝模式"风险。请在部署前仔细阅读以下安全指南。

⚠️

真实安全事件警示

思科 AI 安全研究团队测试了一个第三方 OpenClaw 技能，发现其在用户不知情的情况下进行了数据窃取和提示词注入攻击。在 AI 智能体时代，提示词注入是新型 SQL 注入，务必警惕恶意 Skill 插件。

🔒 最小权限原则

为 OpenClaw 创建专用操作系统用户，禁止访问你的个人 home 目录。不要用主账号的 API Key 和服务凭据。

🐳 容器化隔离

强烈推荐使用 Docker 容器运行 OpenClaw Gateway，限制容器可挂载的目录范围，最小化爆炸半径。

👤 人机协同审批

在配置中启用 require_confirmation: true，所有不可逆操作（删除文件、发送邮件、外部 API 调用）需先征得你的确认。

💰 每日消费上限

设置 daily_spend_limit: 10（$10/天），防止意外的递归任务或错误循环消耗大量 API 费用。

🧩 谨慎安装第三方 Skill

只安装 ClawHub 官方认证或开源可审计的技能插件，对要求访问敏感权限的插件保持高度警惕。

📝 审计日志

启用详细的操作日志记录，定期检查 OpenClaw 的行为日志，发现异常行为及时禁用相关技能或权限。
